3主机管理
3.1纳管主机
接口说明：纳管主机
请保持被纳管主机的用户名和密码一致,如果批量纳管任务失败使用重试功能，请排查纳管失败主机失败原因并进行修复，修复完成以后再进行重试。
3.1.1支持json的接口
请求url：/open_api/insight/external/host/batchAddhost?coverInstall=0
请求类型：POST
入参字段说明：
参数	类型	是否必填	描述
coverInstall	Integer	否	是否覆盖安装，覆盖安装：1；不覆盖安装：0(不传默认为0)
入参示例：
{
　　"coverInstall":0
}

入参字段说明：
参数	类型	是否必填	描述
roomName	String	是	机房名称
ip	String	是	被纳管主机ip（可配置一个或多个ip（多个ip之间用英文逗号分隔）；连续ip使用-连接，起始IP-同网段截止IP中最后一段：例如10.0.0.11-20，代表10.0.0.11到10.0.0.20内所有连续ip）
sshPort	Integer	是	SSH端口（范围：1-65535）
superUser	String	是	用户名（用户名由1-32位数字、字母或下划线组成）
superPwd	String	是	密码（base64加密）
installPath	String	是	安装目录：installPath即在主机上安装组件时指定安装的路径（目录必须是主机的挂载点目录，且不能是挂载点子目录。其中/dev,/run,/sys/fs/cgroup为无用目录，不能使用。安装目录最小可用空间为20G）
dataPath	String	是	数据目录：dataPath即对于dn组件，可以指定它的数据目录。其他组件不会使用此配置（目录必须是主机的挂载点目录，且不能是挂载点子目录。其中/dev,/run,/sys/fs/cgroup为无用目录，不能使用。支持多个路径，多个路径之间用|分隔，数据目录最小可用空间为1G）
label	String	否	标签描述
systemParameter	String	是	是否执行系统参数：0-否，1-是。设置是否执行系统参数是指设置主机服务器的操作系统内核参数，符合GoldenDB运行的要求。如果局方要求使用自己的内核参数，则由局方自行设定，该参数填0。

入参示例：
[{
　　"roomName":"南京机房1",
　　"ip":"10.229.32.71",
　　"sshPort":22,
　　"superUser":"root",
　　"superPwd":"xxxx",
　　"installPath":"/home",
　　"dataPath":"/home|/boot",
　　"label":"xxxx",
　　"systemParameter":"1"
},
{
　　"roomName":"南京机房1",
　　"ip":"10.229.32.72",
　　"sshPort":22,
　　"superUser":"root",
　　"superPwd":"xxxx",
　　"installPath":"/home",
　　"dataPath":"/home|/boot",
　　"label":"xxxx",
　　"systemParameter":"1"
}]

3.2查询纳管主机进度
接口说明：纳管主机是异步接口，利用纳管主机返回taskId可以查询主机纳管进度
请求url：/open_api/insight/external/host/querybatchAddHostResult?taskId=1
请求类型：GET
入参字段说明：
参数	类型	是否必填	描述
taskId	Integer	是	纳管主机传回的任务Id
入参示例：
{
　　"taskId":1
}
出参字段说明：
参数	类型	描述
code	Integer	1：成功；2：失败
msg	String	成功或者失败的描述信息
data	Object	失败时的具体信息

出参示例：
{
　　"code":1,
　　"msg":"success",
　　"data":{
　　　"result":"fail",
　　　"failedIp": ["10.229.43.205","10.229.45.203"],
　　　"installStatus":40,
　　　"errorCode":1001,
　　　"errorMsg":"Failed to manage the host. Please refer to the log for details:/home/goldendb..."
　　}
}
data结构说明：
参数	类型	描述
result	String	success：纳管主机成功；fail：纳管主机失败；running:纳管主机进行中
failedIp	String	发生异常的ip列表
installStatus	Integer	纳管进度
errorCode	Integer	纳管失败错误码,纳管进行中、成功不返回
errorMsg	String	纳管失败错误信息，纳管进行中、成功不返回

3.3查询主机服务器信息
接口说明：查询主机服务器相关信息
请求url：/open_api/insight/external/host/queryHost?ipList=&hostTagList=test1&roomName=&pageNum=1&pageSize=2
请求类型：GET
入参字段说明：
参数	类型	是否必填	描述
ipList	Array	否	当传入的ip列表不为空时，仅返回指定ip的服务器信息；当传入的ip列表为空时，返回全部服务器信息。ipList与hostTagList二选一，都不出现查询所有服务器；都出现以ipList为准，忽略hostTagList。
hostTagList	Array	否	当传入的标签列表不为空时，仅返回拥有标签列表中的某个标签的服务器，当传入的标签列表为空时，返回全部服务器信息。
roomName	String	否	机房名称。指定机房名称，就返回该机房的主机信息。
为空，返回所有机房的主机。
同时指定了ipList或hostTagList，与它们取交集。
pageNum	String	是	当前页
pageSize	String	是	翻页步长

入参示例：
{
　　"ipList": ["ip1","ip2"],
　　"hostTagList": ["abc","edf"],
　　"roomName": "",
　　"pageNum":"1" ,
　　"pageSize": "10"
}
出参字段说明：
参数	类型	描述
code	Integer	1：成功；其他：失败
msg	String	成功或者失败的描述信息
data	Array	主机信息集合
duration	String	请求接口耗时,ms
operationId	String	操作id
出参示例：
{
    "code":1,
    "msg":"success",
    "data":{
        "hostList":[
            {
                "hostId":2,
                "hostIp":"10.229.31.223",
                "hostName":"rh223",
                "roomName":"南京机房2",
                "hostTag":"test1,test2",
                "status":2,
                "cpuUsage":44.15,
                "totalCpu":8,
                "occupiedCpuCores":6,
                "totalMem":57,
                "occupiedMemorySize":12,
                "memUsage":52.81,
                "totalDisksSize":271,
                "usedDiskSize":81,
                "mountPointList":[
                    {
                        "hostIp":"10.229.31.223",
                        "mountPoint":"/",
                        "mountPointTotalSize":50,
                        "mountPointUsedSize":6
                    },
                    {
                        "hostIp":"10.229.31.223",
                        "mountPoint":"/home",
                        "mountPointTotalSize":221,
                        "mountPointUsedSize":75
                    }
                ]
            }
        ],
        "pageInfo":{
            "pages":1,
            "totalRecord":2
        }
    },
    "duration":"50",
    "operationId":"1700623811263"
}
data中字段说明:
hostList:
参数	类型	描述
hostId	Integer	host_info表中的id
hostIp	String	主机ip
hostName	String	主机名
roomName	String	机房名
hostTag	String	标签值
status	Integer	主机状态，0:准备中 1:待使用 2:运行中 3:已下线 4:闲置中
cpuUsage	Double	cpu总使用率%，两位小数。（直接读取host_info表中数据，为机器实际cpu使用率）
totalCpu	Integer	总核数，单位：核
occupiedCpuCores	Integer	已分配cpu核数，指cgroup已分配的cpu核数
totalMem	Integer	总内存大小，单位：GB
occupiedMemorySize	Integer	已分配内存大小，指cgroup已分配的内存
memUsage	Double	内存使用率,%（直接读取host_info表中数据，为机器实际内存使用率）
totalDisksSize	Integer	全部硬盘大小，单位：GB，通过对挂载点空间大小求和获得。挂载点只统计实际磁盘的分区，去掉iso、loop、tmpfs、devpts等虚拟盘。
usedDiskSize	Integer	已使用磁盘大小，单位：GB。上面求和的磁盘大小中已使用的量。（仅挂载点已使用，求和）
mountPointList	Array	挂载点信息

mountPointList字段说明：
参数	类型	描述
hostIp	String	主机ip
mountPoint	String	挂载点路径
mountPointTotalSize	Integer	挂载点磁盘容量，不足1GB会显示为0，单位G
mountPointUsedSize	Integer	挂载点已使用大小，不足1GB会显示为0，单位G

pageInfo字段说明：
参数	类型	描述
pages	Integer	总页数
totalRecord	Integer	总记录数
