鉴权说明
本文所有接口入参需要额外加上操作者的用户名、密码，以实现鉴权功能。在请求头上加username、password字段。
入参字段说明：
参数	类型	是否必填	描述
username	String	是	insight登录用户名
password	String	是	insight登录用户密码，base64加密
postman入参示例：

操作用户由Insight的“权限管理”进行管理。可以添加、删除用户，设置用户的操作权限、数据权限。
操作权限：是否能执行某接口的权限。例如有新增CN的权限，但无删除权限。
数据权限：是否拥有目标实例的权限。假设有租户A和租户B，如果拥有租户A但不拥有租户B的权限，则只能对租户A做操作，对租户B则无权限。
